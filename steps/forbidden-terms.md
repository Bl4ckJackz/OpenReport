# Forbidden Terms — termini che NON devono mai apparire nell'output

La relazione deve leggersi come scritta dall'autore umano. Ogni termine sotto va eliminato/riscritto prima di consegnare il file.

## 1) Riferimenti a sistemi AI / autoassist

Cerca con grep case-insensitive sui file `.md` e `.tex` finali:

```
Claude|Anthropic|Sonnet|Opus|Haiku|GPT|OpenAI|Gemini|Mistral|Llama|Cohere|DeepSeek
LLM|chatbot|AI assistant|AI-assisted|generated by AI|AI-generated
modello linguistico|intelligenza artificiale generativa
🤖 Generated|Co-Authored-By: Claude|Generated with Claude Code
Come AI|In qualità di assistente|As an AI|I am an AI|I was asked to write
```

Se trovati fuori da contesto legittimo (es. tesi su LLM che cita il termine come argomento di studio), riscrivi.

## 2) Tic stilistici tipici di output AI ("AI tells") — IT

Riscrivi sempre quando trovati:

| Espressione AI | Sostituzione consigliata |
|---|---|
| "in conclusione" | "infine", "concludendo", oppure rimuovi |
| "rappresenta una pietra miliare" | "è un passaggio importante" / specifico |
| "navigare nel complesso panorama" | descrivi concretamente il contesto |
| "è importante notare che" | rimuovi (di solito ridondante) |
| "vale la pena sottolineare" | rimuovi o sostituisci con dato concreto |
| "in un'epoca caratterizzata da" | inizio diretto |
| "approfondire" (overuse) | "analizzare", "studiare", "esaminare" |
| "delineare" (overuse) | "descrivere", "presentare" |
| "garantire un'esperienza ottimale" | descrivi cosa offre concretamente |
| "soluzione all'avanguardia" / "stato dell'arte" senza fonte | rimuovi o cita |
| "in un mondo sempre più digitale/connesso" | inizio diretto |
| "abbracciare il cambiamento" | descrivi cosa si è cambiato |
| "non solo... ma anche..." (overuse) | semplifica |
| "in definitiva" (overuse) | usa solo una volta a fine documento |
| "fondamentalmente" (riempitivo) | rimuovi |

## 3) AI tells — EN

| Phrase | Replace with |
|---|---|
| "delve into" | "examine", "study", "look at" |
| "tapestry of" | concrete description |
| "leverage" (verb) | "use" |
| "in conclusion" | "finally", or remove |
| "it is worth noting" | remove (usually redundant) |
| "navigate the complex landscape" | describe concretely |
| "in the realm of" | "in", "within" |
| "embark on a journey" | "start", "begin" |
| "stands as a testament" | concrete claim with evidence |
| "in today's digital age" | direct opening |
| "groundbreaking" / "revolutionary" without source | remove or cite |
| "robust solution" | specifics |
| "foster" (overuse) | "support", "encourage" |
| "comprehensive" (overuse) | specifics |
| "seamlessly" (overuse) | remove or describe |

## 4) Marker grafici sospetti

- Emoji nei titoli accademici (✓, ★, ⚡, 🚀) — rimuovi salvo richiesta utente
- "—" usato dove "," basterebbe (overuse di em-dash) — limita
- Liste a tre elementi parallele e simmetriche ovunque (pattern AI) — varia
- Ogni paragrafo che inizia con "Inoltre,"/"Tuttavia,"/"Pertanto," in sequenza — varia

## 5) Commenti LaTeX da pulire

Rimuovi qualsiasi commento `% ...` che menzioni:
- "AI generated"
- "Claude said"
- "TODO from AI"
- "Generated automatically"

Anche se "solo commenti", finiscono nel sorgente consegnato.

## 6) Self-check command

Lo script `scripts/quality/forbidden-check.sh <file.md|file.tex>` esegue tutti i grep di questo file e ritorna exit-code 1 se trova qualcosa, con elenco occorrenze + numero riga.

Va lanciato:
- Prima dello Step 6 (write output finale)
- Prima dello Step 7 (export)
- Sempre dopo refinement loop di Step 5

## 7) Eccezione legittima

Se la **tipologia stessa** richiede di discutere AI/LLM come oggetto (es. tesi "Valutazione di GPT-4 su task di traduzione"), i termini possono apparire come **soggetto della trattazione**, mai come autoreferenza.

Regola: il documento può **parlare di** Claude/AI; non può **essere firmato da** Claude/AI.
